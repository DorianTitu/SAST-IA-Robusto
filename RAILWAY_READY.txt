ARTHEON SAST - RAILWAY READY
=============================

Tu proyecto está completamente listo para deployar en Railway.

ESTRUCTURA RAILWAY
==================

Root del proyecto tiene:
✓ Dockerfile (para Railway)
✓ railway.json (configuración Railway)
✓ .dockerignore (archivos a ignorar)
✓ requirements.txt (dependencias Python)
✓ src/artheon_sast/ (código de la app)

PASOS PARA DEPLOYAR EN RAILWAY
==============================

1. Ir a railway.app
2. Hacer login / registrarse
3. Click "New Project"
4. Seleccionar "Deploy from GitHub"
5. Autorizar y seleccionar este repositorio
6. Railway detectará el Dockerfile automáticamente
7. Click "Deploy"
8. Esperar a que termine la build

CARACTERÍSTICAS IMPLEMENTADAS
=============================

✓ Puerto dinámico (usa $PORT de Railway)
✓ Dockerfile optimizado en raíz
✓ .dockerignore para builds rápidas
✓ railway.json con configuración completa
✓ Healthcheck endpoint en /health
✓ Logging a stdout para Railway

CÓMO FUNCIONA
=============

1. Railway detecta Dockerfile en raíz
2. Construye imagen Docker
3. Instala dependencias de requirements.txt
4. Copia el código
5. Inicia con: uvicorn src.artheon_sast.api.main:app --host 0.0.0.0 --port $PORT
6. Expone en https://[tu-dominio].railway.app

NOTA IMPORTANTE
===============

El Puerto es DINÁMICO en Railway. Por eso:
- Dockerfile usa: ${PORT:-8000}
- railway.json especifica: $PORT

Esto permite que Railway asigne el puerto que quiera
y la app responda correctamente.
